# AI DB 설계 툴 기획 정리

---

## 1. 툴 아이디어 개요

AI를 활용한 DB 설계 툴로, 아래 4가지 방향을 검토했다.

1. **자연어 → ERD 생성기** ← 가장 임팩트가 큰 방향
2. **DDL 분석 및 개선 제안 툴**
3. **요구사항 문서 → DB 스키마 변환기**
4. **DB 설계 협업 + AI 리뷰 툴**

> 시장성: 자연어 → 스키마가 가장 매력적
> 실무 활용도: DDL → 문서 자동화가 가장 현실적
> 두 개를 하나의 툴로 묶으면 시너지가 높음

---

## 2. 핵심 구조 - 스키마 JSON

스키마 JSON은 이 툴의 **Single Source of Truth** 다.
모든 출력물은 이 JSON에서 파생된다.

```json
{
  "tables": [
    {
      "name": "users",
      "columns": [
        { "name": "id", "type": "BIGINT", "pk": true },
        { "name": "email", "type": "VARCHAR(255)", "unique": true },
        { "name": "created_at", "type": "DATETIME" }
      ]
    }
  ]
}
```

```
스키마 JSON
    ├── ERD 시각화
    ├── DDL 생성
    ├── 테이블 정의서 (문서)
    └── JPA Entity 코드 등
```

---

## 3. AI 응답 형식

AI는 항상 아래 형식의 JSON으로만 응답하도록 강제한다.

```json
{
  "message": "사용자에게 보여줄 메시지 또는 추가 질문",
  "schema": { ... }
}
```

- `message` → 채팅 UI에 출력
- `schema` → 클라이언트 상태를 덮어씌우고 ERD 리렌더링

---

## 4. 진입점 1 - 자연어 대화

### 흐름

```
사용자 자연어 입력
    ↓
AI가 엔티티 추출 및 스키마 생성
    ↓
불명확한 부분은 AI가 능동적으로 질문
    ↓
사용자 답변 → 스키마 수정 반복
    ↓
스키마 완성
```

### 요청 구조

```
시스템 프롬프트
    + 현재 스키마 JSON (전체, 최신 1개)
    + 대화 히스토리 (최근 N턴, 텍스트만)
    + 사용자 입력
```

### 통신 흐름

```
클라이언트                              AI
    │                                   │
    │  [시스템 프롬프트]                 │
    │  [현재 스키마 JSON - 전체]         │
    │  [최근 N턴 대화 히스토리]          │
    │  [사용자 입력]  ─────────────────>│
    │                                   │
    │       { message, schema } <───────│
    │                                   │
    │  ┌─ message → 채팅 출력           │
    │  └─ schema  → 상태 업데이트       │
    │              → ERD 리렌더링       │
    │                                   │
    │  (다음 요청에 새 스키마 포함) ────>│
```

### 페이로드 최적화

| 항목 | 전략 | 이유 |
|---|---|---|
| 스키마 | 매번 전체 전송 | AI가 관계 전체를 파악해야 하므로 |
| 대화 히스토리 | 최근 N턴만 유지 | 오래된 대화는 스키마에 이미 반영됨 |
| 히스토리 내 스키마 | 포함하지 않음 | 중복 누적으로 토큰 낭비 |

> 스키마 JSON은 테이블 20~30개 기준으로도 토큰 수백 개 수준이라 매번 전체를 넘겨도 부담이 적다.
> 페이로드 최적화의 핵심은 **대화 히스토리 관리**다.

## 5. 진입점 2 - 요구사항 문서 업로드

### 지원 문서 형식

| 형식 | 텍스트 추출 | 난이도 |
|---|---|---|
| Word (.docx) | 잘 됨 | 쉬움 |
| PDF (텍스트) | 잘 됨 | 쉬움 |
| PDF (스캔본) | OCR 필요 | 어려움 |
| Excel (.xlsx) | 잘 됨 | 쉬움 |
| Notion / Confluence | 크롤링 or API | 중간 |
| 이미지 (.png, .jpg) | 멀티모달 AI 필요 | 중간 |
| 한글 (.hwp) | 라이브러리 부족 | 어려움 |

> 초기 버전은 **PDF + Word + 텍스트 붙여넣기** 세 가지만 지원 권장
> hwp, 스캔본 PDF는 초기엔 제외하는 것이 현실적

### 스캔본 PDF 처리 방법 (추후 지원 시)

수백 장의 이미지를 AI에게 직접 넘기면 토큰 비용이 폭발적으로 증가한다.

```
스캔본 PDF
    ↓
OCR 선처리로 텍스트 추출 (Tesseract, Google Vision API, AWS Textract 등)
    ↓
텍스트만 AI에게 전달
    ↓
스키마 생성
```

> 멀티모달로 이미지를 직접 넘기는 방식은 비용, 속도, 토큰 한계 문제로 비효율적
> OCR 선처리 → 텍스트 추출 → AI 전달 파이프라인이 가장 합리적

### 프롬프트 구조

```
[시스템 프롬프트]
너는 DB 설계 전문가야.
전달받은 요구사항 문서를 분석해서 아래 규칙을 따라 스키마 초안을 생성해.

규칙:
1. 응답은 반드시 { "message": "...", "schema": { ... } } 형식으로만 해.
2. 문서에 명시되지 않은 내용은 임의로 추가하지 말고 message에서 확인해.
3. 삭제 처리 방식(물리/소프트), 다국어 지원 여부 등 설계에 영향을 주는
   사항이 문서에 없으면 반드시 질문해.
4. 추론한 근거를 message에 간략히 설명해.

[사용자 입력]
아래 요구사항 문서를 분석해서 DB 스키마 초안을 만들어줘.

--- 문서 시작 ---
(추출한 텍스트)
--- 문서 끝 ---
```

### 문서 분석 처리 역할 분담

```
문서 파일
    ↓
텍스트 추출 (코드가 담당)  ← 여기까지만 코드
    ↓
텍스트 → AI에게 전달
    ↓
엔티티/관계 추출 및 스키마 생성 (AI가 담당)  ← 여기부터 AI
```

## 6. 전체 흐름 요약

```
        [진입점 1]              [진입점 2]
        자연어 입력              문서 업로드
            │                       │
            │                  텍스트 추출
            │                       │
            └──────────┬────────────┘
                       ↓
              AI 스키마 초안 생성
                       ↓
              AI 능동적 질문 + 사용자 답변
                       ↓
              스키마 JSON 상태 업데이트
                       ↓
              ERD 시각화 / DDL 출력 / 문서 출력
```

## 7. 추천 기술 스택

### 조합 추천

```
Next.js          - 프론트엔드 + 백엔드 통합 (API Route)
Claude API       - AI (Structured Output, 긴 컨텍스트 윈도우)
React Flow       - ERD 시각화
Vercel AI SDK    - 스트리밍 처리, 대화 히스토리 관리
```

### 각 기술 선택 이유

| 기술 | 이유 |
|---|---|
| Next.js | AI 관련 라이브러리가 React 생태계에 집중, 프론트백 통합으로 생산성 높음 |
| Claude API | Structured Output 지원, 긴 컨텍스트 윈도우, 문서 처리에 유리 |
| React Flow | ERD 시각화 사실상 표준, 커스터마이징 자유로움 |
| Vercel AI SDK | 스트리밍, 히스토리 관리 등 AI 툴 개발에 필요한 기능 내장 |

> 백엔드를 분리하고 싶다면 **FastAPI (Python)** 추천
> PDF 텍스트 추출 등 AI/ML 관련 라이브러리가 Python에 집중되어 있음

---

## 8. 개발 우선순위

1. AI와 대화해서 JSON 스키마 생성 (핵심 기능)
2. JSON → ERD 시각화 연결
3. JSON → DDL 변환기 구현
4. 수정 대화 처리 (컨텍스트 유지)
5. 요구사항 문서 업로드 및 초안 생성
6. 부가 기능 (설계 검증, 다양한 포맷 내보내기 등)

> 1번만 완성해도 이미 유용한 툴이 된다. 나머지는 점진적으로 추가하는 방식이 안전하다.
